## 铁律

1. **`raw/` 只读**：绝对不修改 `raw/` 下任何文件
2. **操作后收尾**：每次操作后更新 `index.meta.toml`，运行 `python3 scripts/build_index.py`，追加 `wiki/log.md`
3. **命名规则**：`概念-[主题].md`（体系性概念）或 `机制-[主题].md`（具体运作规则），禁止自由命名
4. **优先更新**：新资料优先合并进已有页面，不轻易新建
5. **日语例句必附**：所有语法点必须有日语例句 + 中文翻译

## concepts/ 页面规范

### 页面结构（按知识本身组织，不套固定模板）

重写/新建页面时遵循 `prompt/rewrite-prompt.md` 的第 0 步方法论：
1. 拆解所有知识点，标记深度
2. 发现分组逻辑，提出章节结构
3. 章节名来自知识本身（"五段動詞の音便規則"），不用通用模板名（"核心机制"）
4. 深度提炼：碎片→因果链、现象→本质、单点→体系

### 必须包含的要素

- **一句话定义**（blockquote）：说清"解决什么" + 核心关注维度
- **快速导航**：表格或列表，与最终章节标题完全一致
- **日语例句**：每个语法点至少 1 个自然例句 + 中文翻译
- **JLPT 标注**：涉及考试的语法点标注级别（N5-N2）
- **中文母语者误区**：标注母语干扰导致的常见错误
- **内部链接**：`[[页面名]]` Obsidian 风格

## Ingest 流程

1. 读取 `raw/` 指定文件，完整理解
2. 提取核心语法概念、规则、权衡、误区
3. 判断归属层级（L0-L5），优先合并到已有页面
4. 按命名规则落盘
5. 更新 `index.meta.toml`，运行 `build_index.py`，追加 `log.md`

**特定规则**：
- 零散语法点 → 提取背后的语言规则，不照搬列表
- 同一概念多处出现 → 合并到一个 concept 页
- 考试题型 → 提取考点规律，不为每题建页
