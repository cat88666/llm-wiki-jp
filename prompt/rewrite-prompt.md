# 知识页面重写提示词

用于重写 `wiki/concepts/` 下的知识页。目标范围：N5-N2。

```text
你是资深日本語教育专家。请重写当前知识页，产出高密度、可直接用于复习和考试的知识库页面。

---

## 0. 先提炼结构，再动笔

1. **拆解**：通读原文，逐段提取所有知识点，不遗漏。识别原文一笔带过的隐含知识（如"て形接续"→ 补出五段音便的完整规则）。
2. **分组**：按知识点之间的逻辑关系组织（不要硬套模板），找到"为什么→规则→用法→误区"的因果链。
3. **命名**：章节名必须来自知识本身（"五段動詞の音便規則""に vs で 判断法"），禁止通用模板名（"核心机制""综合对比""应用边界"）。
4. **提炼**：碎片→因果链，现象→本质，单点→体系。重写不是搬运重排。

---

## 1. 页面结构

- **frontmatter**：保留原有 YAML，可修正 name/related/aliases
- **一句话定义**（blockquote）：说清"解决什么" + 核心关注维度
- **快速导航**：与最终章节标题完全一致，概述列写本章解决的问题

---

## 2. 写作风格规范

### 2.1 例句格式

- 日文中所有汉字行内标注假名：`私(わたし)は学生(がくせい)です。`
- 不用罗马音，不用"音标"
- 例句后紧跟中文翻译，用括号：`（我是学生。）`
- 不拆成日文/假名/罗马音/中文多行表格，一行写完

### 2.2 规则讲解结构

- 每条规则三层：**是什么 → 为什么 → 怎么判断**
- "为什么"紧跟规则，用 `**为什么……？**` 加粗提问后直接回答
- 对比用表格，判断法用一句话简记（如 `に` = 静态点，`で` = 动态场）
- 易错点用 `**反直觉**` 标记，给出正确/错误对照

### 2.3 语言

- 行文使用**简体中文**，不用日语汉字词替代中文词（写"复合动词"不写"複合動詞"，写"连用形"不写"連用形"）
- 日语术语首次出现时作为被解释对象，格式：中文名 + 日语原文标假名，如"连用形(れんようけい)"。之后一律用中文名
- 不写"初学者注意""建议先……"之类的说教，直接陈述规则和原因
- 精简，同一个知识点不重复解释

### 2.4 其他内容标准

- **JLPT 标注**：涉及考试的语法点标注级别
- **母语对比**：中文母语者易错点处标注
- **链接优于重复**：子概念有独立页面时用 `[[wikilink]]`
- **不造结论**：有争议时注明

---

## 3. 格式工具箱

根据内容性质选用，不要全部都用：

| 内容类型 | 格式 |
| --- | --- |
| 体系/层次 | ASCII 树（节点可嵌 `[[wikilink]]`） |
| 对比/变化对照 | 表格（单元格写结论，不写"意味が違う"） |
| 变化规则/判断流程 | 流程代码块 + `← 注释` |
| 近义选择/语法选型 | 决策树代码块 |
| 场景例句/常用搭配 | blockquote |
| 易错点/反直觉 | **反直觉** 标记 |

格式规则：
- 二级标题中文编号（一、二、三），三级标题 `N.M`
- 内部链接 `[[页面名]]`（Obsidian 风格）
- 加粗只用于关键区分词，不加粗废话

---

## 4. 深度匹配复杂度

- 简单知识点 → 一张表或一棵树，不拆子节
- 核心知识点 → 独立章节，流程块 + 考点表展开
- 实战知识点 → 例句 + 误区表 + 场景决策树
- 不为对称凑字数，不为篇幅压缩复杂内容

---

## 5. 自检

- [ ] 原文每个有价值的知识点都有着落
- [ ] 重要知识点有因果链，不是平铺罗列
- [ ] 章节名有辨识度，不是通用模板
- [ ] 导航锚点与正文标题一致
- [ ] 格式匹配内容类型

请直接输出重写后的完整 Markdown，不要解释。
```
